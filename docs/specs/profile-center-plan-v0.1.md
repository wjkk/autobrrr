# Profile Center Plan v0.1

## 目标

为 `/profile` 建立一个明确的个人中心入口，不再让顶部 Profile 按钮跳转到不存在页面，并逐步把“账号、偏好、点数、通知、安全”收口到同一处。

## 当前代码现状

- 已有入口：
  - `/my-space`
  - `/settings/providers`
  - `/settings/catalogs`
- 已有顶部 Profile 按钮：
  - `apps/web/src/features/shared/components/system-shell.tsx`
- 当前缺口：
  - 没有 `/profile` 页面
  - 没有统一的账号资料聚合层
  - 点数、通知、登录设备、账单都未在前端形成入口

## 第一阶段

先落一个真实可访问的个人中心页，承担：

1. 账号资料占位
2. 快速跳转到已有设置入口
3. 后续功能规划入口

这一阶段不引入后端新接口，只做前端骨架。

## 推荐功能优先级

### P0

1. 账号信息
   - 头像
   - 昵称
   - 邮箱/手机号
   - 团队与角色

2. 创作偏好
   - 默认内容类型
   - 默认图像模型
   - 默认视频模型
   - 常用风格预设

3. 点数与套餐
   - 当前余额
   - 最近 30 天消耗
   - 消耗来源明细
   - 套餐续费入口

### P1

1. 通知中心
   - 任务完成
   - 任务失败
   - 发布反馈
   - 系统公告

2. 账号安全
   - 登录设备
   - 最近登录记录
   - 第三方授权
   - 风险操作确认

### P2

1. 创作偏好学习
   - 最近使用模型
   - 最近常用主体
   - 最近常用风格
   - 快捷恢复到最近工作流

2. 个人成就与数据概览
   - 累计项目数
   - 累计导出次数
   - 最近产出趋势

## 页面信息架构建议

1. 顶部 Hero
   - 用户身份
   - 当前账号状态
   - 快速入口

2. 中部双栏
   - 左侧：账号/偏好/安全
   - 右侧：点数/通知/最近活动

3. 底部扩展区
   - 产品路线
   - FAQ
   - 支持入口

## 后续实现建议

1. 先补 profile bootstrap server API
2. 再接入真实 user / points / notifications 数据
3. 最后把 profile 和通知、账单、安全页拆成子路由
